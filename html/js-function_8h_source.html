219 // Tiering up a function happens as follows:
220 // 1. RequestOptimization is called
221 // -> From now on `IsOptimizationRequested` and also
222 // `IsTieringRequestedOrInProgress` return true.
223 // 2. On the next function invocation the optimization is triggered. While the
224 // optimization progresses in the background both
225 // `IsTieringRequestedOrInProgress` and `tiering_in_progress` return
226 // true. It also means the optimization is no longer requested (i.e.,
227 // `IsOptimizationRequested` returns false).
228 // 3. Once the compilation job is finalized the functions code is installed
229 // via `UpdateCode` and any remaining flags cleared by
230 // `ResetTieringRequests`.
231 // NB: Osr tiering state is tracked separately from these.
232
233 // Mark this function for optimization. The function will be recompiled
234 // the next time it is executed.
