38 // Returns true if a holder contains an element with the specified index
39 // without iterating up the prototype chain. The first version takes the
40 // backing store to use for the check, which must be compatible with the
41 // ElementsKind of the ElementsAccessor; the second version uses
42 // holder->elements() as the backing store. If a |filter| is specified,
43 // the PropertyAttributes of the element at the given index are compared
44 // to the given |filter|. If they match/overlap, the given index is ignored.
45 // Note that only Dictionary elements have custom

214 // Element handlers distinguish between entries and indices when they
215 // manipulate elements. Entries refer to elements in terms of their location
216 // in the underlying storage's backing store representation, and are between 0
217 // and GetCapacity. Indices refer to elements in terms of the value that would
218 // be specified in JavaScript to access the element. In most implementations,
219 // indices are equivalent to entries. In the NumberDictionary
220 // ElementsAccessor, entries are mapped to an index using the KeyAt method on
221 // the NumberDictionary.
