43// An IsolateGroup allows an API user to control which isolates get allocated
44// together in a shared pointer cage.
45//
46// The standard configuration of V8 is to enable pointer compression and to
47// allocate all isolates in a single shared pointer cage
48// (V8_COMPRESS_POINTERS_IN_SHARED_CAGE). This also enables the sandbox
49// (V8_ENABLE_SANDBOX), of which there can currently be only one per process, as
50// it requires a large part of the virtual address space.
51//
52// The standard configuration comes with a limitation, in that the total size of
53// the compressed pointer cage is limited to 4 GB. Some API users would like
54// pointer compression but also want to avoid the 4 GB limit of the shared
55// pointer cage. Isolate groups allow users to declare which isolates should be
56// co-located in a single pointer cage.
57//
58// Isolate groups are useful only if pointer compression is enabled. Otherwise,
59// the isolate could just allocate pages from the global system allocator;
60// there's no need to stay within any particular address range. If pointer
61// compression is disabled, there is just one global isolate group.
62//
63// Note that JavaScript objects can only be passed between isolates of the same
64// group. Ensuring this invariant is the responsibility of the API user.
65class V8_EXPORT_PRIVATE IsolateGroup final {
