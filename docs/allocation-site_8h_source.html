102 // A "zombie" AllocationSite is one which has no more strong roots to
103 // it, and yet must be maintained until the next GC. The reason is that
104 // it may be that in new space there are AllocationMementos hanging around
105 // which point to the AllocationSite. If we scavenge these AllocationSites
106 // too soon, those AllocationMementos will end up pointing to garbage
107 // addresses. The concrete case happens when evacuating new space in the full
108 // GC which happens after sweeping has been started already. To mitigate this
109 // problem the garbage collector marks such AllocationSites as zombies when it
110 // discovers there are no roots, allowing the subsequent collection pass to
111 // recognize zombies and discard them later.
