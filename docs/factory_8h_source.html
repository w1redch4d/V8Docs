255 // String creation functions. Most of the string creation functions take
256 // an AllocationType argument to optionally request that they be
257 // allocated in the old generation. Otherwise the default is
258 // AllocationType::kYoung.
259 //
260 // Creates a new String object. There are two String encodings: one-byte and
261 // two-byte. One should choose between the three string factory functions
262 // based on the encoding of the string buffer that the string is
263 // initialized from.
264 // - ...FromOneByte (defined in FactoryBase) initializes the string from a
265 // buffer that is Latin1 encoded (it does not check that the buffer is
266 // Latin1 encoded) and the result will be Latin1 encoded.
267 // - ...FromUtf8 initializes the string from a buffer that is UTF-8
268 // encoded. If the characters are all ASCII characters, the result
269 // will be Latin1 encoded, otherwise it will converted to two-byte.
270 // - ...FromTwoByte initializes the string from a buffer that is two-byte
271 // encoded. If the characters are all Latin1 characters, the result
272 // will be converted to Latin1, otherwise it will be left as two-byte.
273 //
274 // One-byte strings are pretenured when used as keys in the SourceCodeCache.
275 template <size_t N>
276 inline Handle<String> NewStringFromStaticChars(
277 const char (&str)[N], AllocationType allocation = AllocationType::kYoung);
