307// The CpuProfiler is a sampling CPU profiler for JS frames. It corresponds to
308// v8::CpuProfiler at the API level. It spawns an additional thread which is
309// responsible for triggering samples and then symbolizing the samples with
310// function names. To symbolize on a background thread, the profiler copies
311// metadata about generated code off-heap.
312//
313// Sampling is done using posix signals (except on Windows). The profiling
314// thread sends a signal to the main thread, based on a timer. The signal
315// handler can interrupt the main thread between any abitrary instructions.
316// This means we are very careful about reading stack values during the signal
317// handler as we could be in the middle of an operation that is modifying the
318// stack.
319//
320// The story on Windows is similar except we use thread suspend and resume.
321//
322// Samples are passed to the profiling thread via a circular buffer. The
323// profiling thread symbolizes the samples by looking up the code pointers
324// against its own list of code objects. The profiling thread also listens for
325// code creation/move/deletion events (from the GC), to maintain its list of
326// code objects accurately.
