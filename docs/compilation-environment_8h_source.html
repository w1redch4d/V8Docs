42// The Arm architecture does not specify the results in memory of
43// partially-in-bound writes, which does not align with the wasm spec. This
44// affects when trap handlers can be used for OOB detection; however, Mac
45// systems with Apple silicon currently do provide trapping beahviour for
46// partially-out-of-bound writes, so we assume we can rely on that on MacOS,
47// since doing so provides better performance for writes.
