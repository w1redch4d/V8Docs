181// The number of calls to an exported Wasm function that will be handled
182// by the generic wrapper. Once the budget is exhausted, a specific wrapper
183// is to be compiled for the function's signature.
184// The abstract goal of the tiering strategy for the js-to-wasm wrappers is to
185// use the generic wrapper as much as possible (less space, no need to compile),
186// but fall back to compiling a specific wrapper for any function (signature)
187// that is used often enough for the generic wrapper's small execution penalty
188// to start adding up.
189// So, when choosing a value for the initial budget, we are interested in a
190// value that skips on tiering up functions that are called only a few times and
191// the tier-up only wastes resources, but triggers compilation of specific
192// wrappers early on for those functions that have the potential to be called
193// often enough.
194constexpr uint32_t kGenericWrapperBudget = 1000;
